# product-world-builder

Convert product purpose into a visual metaphor and consistent UI language.

## Process
1. Identify the core transformation: what goes in, what comes out
2. Find the physical analogy (circuit board, control room, terminal, radar, etc.)
3. Define one color (not a palette) that owns the brand
4. Define one typography contrast (e.g. ultra-bold display vs. light mono body)
5. Define one motion signature (draw, reveal, trace, scan, pulse)
6. Apply consistently: every new component asks "does this fit the metaphor?"

## For automateable specifically
- Metaphor: Signal processing / circuit traces
- In: messy manual workflows. Out: clean deterministic systems.
- Color: precise blue (#2563eb) on near-black
- Typography: 900-weight editorial headlines + mono for all data
- Motion: paths that draw, systems that come online
